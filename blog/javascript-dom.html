Share an analogy to describe JavaScript and its relationship to HTML and CSS.


Imagine a typical analogue clock hanging on a wall.

The HTML is analogous to the objects and any writing on the clock. The hands of the clock, the face of the clock, the numbers on the clock.

The CSS is analogous to the style of the clock - the font of the numbers, the colour, the size, the shape. To really get into the detail of the analogy, it even represents the position of each of the numbers around in a circle of the clock, and further, the movement of the clock hands around the face and the ticking sound.

The Javascript is analogous to the interactive mechanisms of the clock. If you flip it upside down there may be a knob or buttons you can use to change the time on the clock. These interfaces that allow you to modify the clock are like JS in web development.

Explain control flow and loops using an example process from everyday life, for example, 'waking up' or 'brushing your teeth' (but not those).

Consider the errand of walking to a vending machine down the hall to buy a bottle of milk.

Our errand would involve the following steps:

1. EXIT office
      2. Take a STEP down the hall
      3. Are you at the VENDING MACHINE? 
        If no: loop back to step 2. If yes, continue.
      4. BUY chocolate bar

Control flow is process of following these steps according to some rules. Just like in a recipe, the rules can be implicit. For example, it is implicit that we read a recipe starting at the top of the page with the number 1, and typically proceed down the page following instructions from lowest number to highest.

However the rules about control can also be explicit within the recipe, typically when the implicit rules are to be varied from. A loop is an example of such a variation. When a loop is initiated, this means the recipe should continue from an earlier step in the recipe, and proceed forward from there.

In our example, the loop is initiated at step 3, and loops back to step 2 until the condition of being at the vending machine is met.

Describe what the DOM is and an example of how you might interact with it.

Imagine you had a weird voodo doll’s house where you could move things around in the dolls house and the things would move for real in the haunted house down the street, but also if things changed in the real house, they would move in the doll’s house.

This would be a DHOM, a “Doll’s house object model”.

The doll’s house is a model of a real house. You can use it to see where objects in the house are. The table is in the dining room. The frying pan is in the kitchen. You can also use the model to move objects around in the real world. Want to put the frying pan object into the dining room? Sure, go ahead and put it on the table. The model changes, the real world changes.

The document object model is basically like this, except the model is a long document saying what objects are in what locations, and the document object model is coupled with a webpage you are building instead of a creepy haunted house.

Explain the difference between accessing data from arrays and objects.

Accessing data from an object is like looking something up in the dictionary. You have to know the word you are looking for and then flip to the corresponding page and retrieve the entry, in this example the meaning of a word.

Accessing data from an array is more like pulling a card from a fan of cards. You can draw from the top, or you can draw the card that is fifth along or 20th along the fan. Once you draw the card, the entry is retrieved, in this case the identify of the card e.g ‘the nine of hearts’.

Explain what functions are and why they are helpful.

A function is a piece of code which is prepared ahead of time to perform some task when you need it to, on the inputs that you need it to perform the tasks on.

This is useful when you have a task that you may need to repeat many times. You can simply call the function rather than rewriting the code.

Use of functions can also reduce the amount of code you need to rewrite if you have to change something in your code.


For example, say you have a function play() that performs 10 tasks on an object. If you were to call play(ball) it would throw the ball then catch it, then bounce it and so on.

What if you wanted to play with a chainsaw instead, you could simply change the code to play(chainsaw) rather than going through all ten commands and changing each one from throw-ball to throw-chainsaw etc.
